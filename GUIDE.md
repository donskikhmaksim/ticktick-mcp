# Как пользоваться этим помощником для TickTick

Это руководство для обычного пользователя — без технических терминов.

## Что это такое

TickTick — это приложение для списков дел и задач. Обычно вы открываете его,
находите нужный список, вручную создаёте задачу, ставите срок, приоритет, тег…
Этот «MCP» — мостик, через который **Claude управляет вашим TickTick за вас**.
Вы просто пишете обычными словами, что хотите, а Claude сам нажимает нужные
кнопки внутри TickTick: добавляет задачи, переносит сроки, отмечает выполненное,
раскладывает дела по проектам и тегам, показывает, что горит.

Проще говоря: **вы говорите — Claude делает это в вашем TickTick.**

## Что он умеет (в двух словах)

- Заводить задачи и целые списки дел, в том числе с подзадачами.
- Ставить и переносить сроки, приоритеты, напоминания и повторения
  («каждый понедельник»).
- Отмечать сделанное, отменять и удалять лишнее (с подтверждением перед
  удалением).
- Показывать нужные срезы: что на сегодня, что просрочено, что на неделю,
  что важное.
- Раскладывать дела по проектам (спискам) и папкам, вешать теги.
- Вести привычки, оставлять комментарии к задачам, прикреплять файлы.
- Показывать статистику и историю изменений.

## Когда этим пользоваться

- Когда лень открывать приложение и кликать — быстрее написать словами.
- Когда нужно навести порядок: разобрать входящие, разложить по проектам,
  почистить просроченное.
- Когда хочется быстрый обзор: «что у меня сегодня?», «что я не успел?».
- Когда задача приходит в разговоре — можно сразу попросить занести её.

## Примеры: что написать и что произойдёт

Просто пишите Claude обычными фразами. Вот типичные просьбы и что за ними
следует.

| Вы пишете | Что происходит |
| --- | --- |
| «Добавь задачу купить молоко на завтра с высоким приоритетом» | В TickTick создаётся задача «Купить молоко» со сроком на завтра и высоким приоритетом. |
| «Покажи, что просрочено» | Claude показывает список всех задач, у которых срок уже прошёл. |
| «Что у меня на сегодня?» | Показывается список задач со сроком на сегодняшний день. |
| «Перенеси дедлайн встречи с юристом на пятницу» | Claude находит задачу про встречу и меняет ей срок на ближайшую пятницу. |
| «Отметь, что я оплатил счёт за интернет» | Задача про оплату интернета отмечается выполненной (галочка). |
| «Заведи проект „Ремонт“ и накидай туда: купить краску, вызвать электрика, заказать двери» | Создаётся новый проект «Ремонт», и в него добавляются три задачи. |
| «Сделай задачу „Зарядка“ повторяющейся каждый будний день в 7 утра» | К задаче добавляется повтор по будням с напоминанием на 7:00. |
| «Перенеси все задачи из Входящих в проект „Работа“» | Задачи из папки «Входящие» перемещаются в проект «Работа». |
| «Повесь тег „срочно“ на задачу про отчёт» | На задачу про отчёт вешается метка «срочно». |
| «Удали задачу „позвонить в банк“» | Claude сначала показывает, что именно удалит, и удаляет только после вашего подтверждения. |
| «Как у меня с привычкой „пить воду“ на этой неделе?» | Показывается серия отметок и прогресс по привычке. |
| «Дай короткий обзор: что важного на неделю» | Claude показывает задачи с приоритетом и сроком на ближайшие 7 дней. |

Не нужно помнить точные названия — пишите как удобно, Claude сам подберёт
подходящее действие.

> Этот раздел собирается автоматически из кода сервера (`scripts/gen_guide.py`). Руками не редактируйте — при следующем запуске генератора правки затрутся.

**Всего инструментов: 70.**

### Задачи (создать, изменить, выполнить, удалить)

| Инструмент | Что делает |
| --- | --- |
| `create_tasks` | Создать одну или несколько задач, при желании сразу с подзадачами. |
| `create_subtask` | Добавить подзадачу (пункт) внутри уже существующей задачи. |
| `update_tasks` | Изменить задачу: название, срок, приоритет, заметку и т. д. |
| `complete_tasks` | Отметить задачи выполненными (поставить галочку). |
| `abandon_task` | Отметить задачу как «не буду делать» (отменить, не удаляя). |
| `delete_tasks` | Удалить задачи насовсем (в корзину). |
| `delete_task_with_subtasks` | Удалить задачу вместе со всеми её подзадачами разом. |
| `duplicate_task` | Сделать копию задачи в том же проекте. |
| `move_tasks` | Перенести задачи в другой список/проект. |
| `set_task_parent` | Вложить задачи внутрь другой задачи (сделать их подзадачами). |
| `unset_task_parent` | Вынуть подзадачу наружу, сделав её обычной задачей. |
| `attach_file_to_task` | Прикрепить файл к задаче (по ссылке или содержимому). |
| `plan_task_creation` | Шаг 1 безопасного создания: показать, что именно будет создано, до подтверждения. |
| `execute_task_creation` | Шаг 2 безопасного создания: создать ровно то, что было показано в плане. |
| `plan_task_deletion` | Шаг 1 безопасного удаления: показать, что именно удалится, до подтверждения. |
| `execute_task_deletion` | Шаг 2 безопасного удаления: удалить ровно то, что было в плане. |
